Phi

A statically typed, purely functional language with Haskell-style syntax — type classes, do-notation, pattern matching — that compiles directly to Erlang BEAM bytecode. Written in Rust.

module HelloWorld (main) where import System.IO (println) main :: IO () main = println "Hello World from Phi!"

Features

Power & Beauty

Write pure functional code. Compile to highly scalable, fault-tolerant Erlang bytecode.

Full Type Safety

Compile-time type checking and inference. Algebraic data types, pattern matching, type classes.

Erlang Concurrency

Native receive/after syntax, message passing, and actors — all targeting the battle-tested BEAM runtime.

Getting Started

# Requirements: Rust 1.85+, Erlang/OTP 25+
$ cargo build --release

# Compile stdlib + test modules to ebin/
$ cargo run -- src/stdlib src/tests
--- Batch Parsing ---
Summary: 194/194 files parsed successfully.

--- Typechecking ---
Typechecking done: 194/194 ok

--- Codegen (.phi → .beam directly) ---
  Direct .beam: 194 | failed: 0

# Run compiled modules in Erlang
$ erl -pa ebin
> F = 'Phi.HelloWorld':main(), F().
Hello World from Phi!

Properties

• Hindley-Milner type inference with type class constraints
• Algebraic data types (ADT)
• Type classes — class/instance/where, dictionary passing
• Pattern matching — constructors, literals, tuples, list patterns, guards
• Do-notation and monadic >>= / >> / pure
• List comprehensions — [f x | x <- xs, pred x]
• receive/after syntax for Erlang message passing
• Where-clauses and local let bindings
• Lambda lifting with free-variable capture (make_fun3)
• Foreign Function Interface (foreign import)
• 194 stdlib modules — Data.List, Data.Map, Control.Monad, System.IO, and more
• Compiles directly to BEAM bytecode — no erlc involved
• Parallel compilation via rayon
